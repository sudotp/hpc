PRACTICAL 1 - Parallel BFS and DFS using OpenMP
CODE EXPLANATION + OUTPUT EXPLANATION + TEACHER Q&A
================================================================

WHAT THIS PRACTICAL DOES
--------------------------
Implements BFS and DFS graph traversal on a 7-node undirected
graph using OpenMP for parallel neighbor exploration.

Graph used:
      0
     / \
    1   2
   / \   \
  3   4   5
           \
            6

================================================================
CODE EXPLANATION
================================================================

#include <omp.h>
  Includes OpenMP library for parallel programming.

const int NODES = 7;
  Total nodes in the graph = 7 (numbered 0 to 6).

vector<int> graph[NODES];
  Adjacency list — graph[i] stores all neighbors of node i.
  graph[0] = [1,2], graph[1] = [0,3,4], etc.

addEdge(u, v):
  Adds edge between u and v in BOTH directions (undirected).
  graph[u] gets v, graph[v] gets u.

--- BFS FUNCTION ---

vector<bool> visited(NODES, false);
  All nodes start as unvisited (false).

queue<int> q;
  Queue stores nodes to visit — FIFO (First In First Out).

visited[start]=true; q.push(start);
  Mark node 0 as visited and add to queue to begin.

while(!q.empty()):
  Keep running until all reachable nodes are visited.

currentLevel = all nodes at current BFS level.
  We take ALL nodes at one level, print them, then explore
  their neighbors in parallel.

#pragma omp parallel for
  Splits the currentLevel loop across multiple threads.
  Each thread handles a different node simultaneously.

#pragma omp critical
  Only ONE thread can enter this block at a time.
  Prevents two threads marking same neighbor simultaneously
  (Race Condition).

--- DFS FUNCTION ---

vector<int> stack;
  Stack stores nodes — LIFO (Last In First Out).
  We use iterative DFS (not recursive) to avoid deadlock.

stack.back() / stack.pop_back()
  Takes the TOP element from stack and removes it.

#pragma omp parallel for + critical
  All neighbors checked in parallel.
  Unvisited ones added to toVisit list safely.

for(int nb : toVisit) stack.push_back(nb);
  After parallel check, push found neighbors to stack.

--- MAIN ---

addEdge calls build the graph with edges:
  0-1, 0-2, 1-3, 1-4, 2-5, 5-6

parallelBFS(0) — starts BFS from node 0.
parallelDFS(0, dfsVisited) — starts DFS from node 0.


================================================================
OUTPUT EXPLANATION
================================================================

Output:
  Graph has 7 nodes (0 to 6)
  Starting node: 0
  Parallel BFS traversal: 0 1 2 3 4 5 6
  Parallel DFS traversal: 0 2 5 6 1 4 3

WHY BFS gives: 0 1 2 3 4 5 6
  BFS visits level by level:
  Level 0: 0           (start)
  Level 1: 1 2         (neighbors of 0)
  Level 2: 3 4 5       (neighbors of 1 and 2)
  Level 3: 6           (neighbor of 5)
  Result: 0 1 2 3 4 5 6 — always in this order.

WHY DFS gives: 0 2 5 6 1 4 3
  DFS uses a stack — goes deep first.
  Start: push 0 → pop 0, push neighbors [1,2]
  Pop 2 (top of stack), push its neighbor [5]
  Pop 5, push [6]
  Pop 6, no new neighbors
  Pop 1, push [3,4]
  Pop 4, pop 3
  Result: 0 2 5 6 1 4 3
  Note: DFS order can vary slightly due to parallelism —
  this is CORRECT and EXPECTED behavior.

We did NOT manually set these values.
The algorithms produced this order automatically.


================================================================
TEACHER QUESTIONS & ANSWERS
================================================================

Q. What is BFS?
A. Visits nodes level by level using a Queue (FIFO).

Q. What is DFS?
A. Goes deep first using a Stack (LIFO), backtracks when stuck.

Q. What is #pragma omp parallel for?
A. Splits loop iterations across multiple threads to run in parallel.

Q. What is #pragma omp critical?
A. Only one thread runs this block at a time. Prevents race conditions.

Q. Why iterative DFS instead of recursive?
A. Recursive DFS + omp critical causes deadlock. Iterative stack avoids it.

Q. What is a Race Condition?
A. Two threads access same data simultaneously causing wrong results.
   Critical section prevents this.

Q. What is an Adjacency List?
A. Each node stores a list of its neighbors. graph[0]=[1,2] means
   node 0 connects to 1 and 2.

Q. Why does DFS output differ from BFS?
A. BFS uses Queue (level by level). DFS uses Stack (deep first).
   Different data structures give different traversal orders.

Q. What does visited[] do?
A. Tracks which nodes are already seen. Prevents infinite loops.

Q. Compile command?
A. g++ -fopenmp P1_Parallel_BFS_DFS.cpp -o bfs_dfs
